Jira KT Bot

An internal AI agent built for the Clarios IT and development teams to query Jira project data using natural language. Instead of navigating Jira manually, team members can ask the bot questions and get structured, accurate results pulled directly from indexed project data.

Role UX/UI Applications & Data Analytics Intern
Company Clarios
Users IT Team & Developers
Microsoft Copilot Studio Microsoft Graph Connector Jira Microsoft 365

Overview

The Jira KT Bot (Knowledge Transfer Bot) is an internal Copilot Studio agent built to help Clarios IT teams and developers access Jira project data through natural language conversation. The goal was to reduce the friction of manually searching through Jira tickets and make project information easier to surface during knowledge transfer and day-to-day work.

Rather than building the agent with the standard Power Platform Jira connector - which introduced payload size limitations and reliability issues - I used a Microsoft Graph connector to index Jira data directly into Microsoft 365. This approach gave the agent a much more reliable and scalable knowledge source, with results automatically scoped to what each user has permission to see in Jira.

The Problem

Manual Jira Navigation

Finding specific tickets, checking statuses, or summarizing project activity required manually searching through Jira - slow and disruptive during meetings or knowledge transfer sessions.

Knowledge Transfer Gaps

When team members needed to get up to speed on a project, there was no fast way to pull a summary of recent activity, open issues, or ticket history without digging through Jira themselves.

Connector Limitations

The standard Power Platform Jira connector hit payload size limits when returning large result sets, making it unreliable for real-world query volumes.

What I Built

01

Graph Connector Knowledge Source

Instead of using the Power Platform Jira connector as a tool inside the agent, I used a Microsoft Enterprise Graph connector to index Jira issues directly into Microsoft 365. This means the agent can semantically search Jira data the same way it searches any other Microsoft 365 knowledge source - without hitting payload limits or needing manual JQL query configuration. The indexed data includes issue keys, statuses, assignees, summaries, dates, and links back to the original Jira ticket.

02

Natural Language Jira Queries

Team members can ask the bot questions in plain English and get structured results back. For example, asking "show me recent Jira tasks from OneVarta With Sitefinity" returns a formatted list of the most recent tickets with issue key, status, assignee, summary, dates, and a direct link back to Jira. No JQL knowledge required.

03

Permission-Scoped Results

Because the data is indexed through the Microsoft Graph connector, the agent automatically respects each user's existing Jira permissions. Users only see issues they already have access to in Jira - no additional access control configuration needed inside the agent itself.

04

Generative Orchestration Design

The agent was designed around Copilot Studio's generative orchestration mode rather than rigid predefined topic flows. This means the agent reasons about what the user is asking and decides when to call the Jira knowledge source versus responding from general context - making conversations feel natural and flexible rather than scripted.

Technical Details

Why Graph Connector Over Power Platform Connector

The Power Platform Jira connector was the first approach tried, but it ran into payload size issues when returning realistic result sets. The Microsoft Graph connector indexes Jira Cloud issues into Microsoft 365 at the admin level, making them searchable across the Microsoft 365 ecosystem. This approach is more reliable, handles larger data volumes, and integrates cleanly with Copilot Studio as a knowledge source rather than a tool action.

What the Agent Returns

For each Jira query the agent surfaces: issue key, status, assignee, full summary, created and closed dates, and a direct link back to the ticket in Jira. Results are formatted cleanly in the chat interface so they are easy to scan without leaving the conversation.
