WordPress Cloud Deployment with AWS Lightsail & CDN

Set up a WordPress website from scratch using AWS — from registering the domain and configuring the server, to adding security certificates, setting up cloud storage for media files, and using a content delivery network for fast page loads worldwide.

Type Personal Project
AWS Lightsail Route 53 CloudFront CDN Amazon S3 AWS Certificate Manager IAM WordPress

Project Overview

This project is a real-world WordPress website built entirely on AWS. The goal was to create a fast, secure, and affordable site using cloud services — and to learn how different AWS tools work together in a production setup.

Instead of traditional web hosting, I used AWS Lightsail for the server, Route 53 for the domain, S3 for storing images and media, CloudFront for fast delivery worldwide, and Certificate Manager for HTTPS security. The result is a professional website that's secure, fast, and easy to maintain.

Architecture Overview

Web Server

AWS Lightsail running WordPress — a simple, affordable way to host a website without managing complex server infrastructure.

Fast Delivery

CloudFront delivers images and files from servers close to each visitor, so the site loads quickly no matter where someone is in the world.

Media Storage & Security

S3 stores all images and media files separately from the server, with proper access controls and HTTPS encryption on everything.

Domain Setup

Route 53 handles the domain name and directs traffic to the right place — the web server for pages, the CDN for media files.

Implementation Phases

01

Set Up the Server

Created a Lightsail instance with WordPress pre-installed. Lightsail is AWS's simpler hosting option — it gives you a working server with predictable monthly pricing. Assigned a static IP address so the domain always points to the right place.

Why Lightsail: For a WordPress site, Lightsail offers the right balance of simplicity and control without the complexity of managing a full EC2 instance.

02

Connect the Domain

Registered the domain through Route 53 and set up DNS records to point the domain to the server. Created separate records for the main site (example.com), the www version, and a CDN subdomain for media delivery.

DNS Records Created:

  • A record: @ → Lightsail static IP
  • A record: www → Lightsail static IP
  • CNAME record: cdn → CloudFront distribution
03

Configure WordPress

Logged into the WordPress admin panel, changed the default credentials for security, and configured the site to use the custom domain. Prepared the setup for the next steps: adding SSL and connecting cloud storage.

04

Add HTTPS Security

Got free SSL certificates through AWS Certificate Manager for the main domain, www subdomain, and CDN subdomain. AWS handles automatic renewal so the certificates never expire without manual intervention.

Why Multiple Certificates: Each subdomain needs its own certificate so that HTTPS works everywhere visitors might access the site or its media files.

05

Set Up Cloud Storage for Media

Created an S3 bucket to store images and media files separately from the web server. Set up a dedicated user account with the right permissions to upload files securely. Installed a WordPress plugin that automatically sends new uploads to S3 and can remove them from the server to save space.

Added the AWS credentials to the WordPress configuration file so the site can communicate with S3 securely. Also set up permissions so CloudFront can read the files for delivery.

Why This Matters: Keeping media files separate from the server means the server runs faster, storage scales automatically, and the site can grow without hitting capacity limits.

06

Add Global Content Delivery

Set up CloudFront to serve media files from servers around the world instead of the single origin server. Configured it to pull from the S3 bucket securely — the bucket itself stays private while CloudFront handles public delivery.

Updated the WordPress plugin to use CloudFront URLs for all media files. This means when someone in Europe loads the site, their images come from a nearby server instead of traveling all the way back to the US.

Speed Impact: CloudFront caches content at locations worldwide, so visitors download images from servers close to them — making pages load noticeably faster for international visitors.

Technical Highlights & Key Decisions

Automatic Media Uploads

Configured a WordPress plugin to automatically send uploaded images to S3 and remove them from the server, keeping the server lean without any manual steps.

Secure Access Controls

Created a dedicated user account for S3 access with only the permissions it needs — rather than using the main admin account. Stored credentials securely in the WordPress config file.

Private Bucket, Public Delivery

The S3 bucket stays private — only CloudFront can read from it. This means files are delivered globally through the CDN without exposing the storage bucket directly.

Six AWS Services Working Together

Connected Lightsail, Route 53, Certificate Manager, S3, IAM, and CloudFront into one working system. Each service handles a specific job while they all work together seamlessly.

Skills Developed

  • Cloud Setup: Learned to design and build cloud infrastructure that's secure, fast, and cost-effective. Understand when to use different AWS services and how they fit together.
  • Networking: Gained practical experience with DNS, domain configuration, and how internet traffic gets routed from a user's browser to the right server.
  • Security: Implemented multiple layers of security — HTTPS certificates, access controls, private storage with public delivery. Learned AWS's approach to shared security responsibility.
  • Automation Mindset: Set up systems that run on their own — automatic certificate renewal, automatic media uploads to cloud storage, and automatic content caching worldwide.
  • Troubleshooting: Developed a systematic approach to diagnosing issues across connected cloud services, from DNS problems to storage permissions.
