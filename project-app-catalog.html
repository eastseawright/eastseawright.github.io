Application Catalog

A company-wide searchable directory for 900+ business applications built with Microsoft Power Apps and SharePoint. Employees across 7 global regions can find the right tool, see ownership details, and request access - all from one place.

Role UX/UI Applications & Data Analytics Intern
Company Clarios
Scale 900+ Apps, 7 Global Regions
Microsoft Power Apps SharePoint Power Fx Power Automate

Overview

Clarios had no central place to view the 900+ applications used across 7 global regions. Multiple apps were duplicating the same capabilities, and nobody had a clear picture of who owned what or where it was available. Teams wasted time searching for the right tool or accidentally building something that already existed.

I designed and built an Application Catalog using Microsoft Power Apps that serves as the single source of truth for all business applications. It lives at the front of the Global Application SharePoint site so anyone in the company can find what they need.

The Problem

No Single Source of Truth

Application data was scattered across spreadsheets and systems. Nobody had a complete picture of what tools were available or who was responsible for them.

Redundant Software Spend

Without visibility, teams were building or buying tools that already existed elsewhere in the organization.

No Clear Access Path

Employees had no way to know how to request access to a tool, who to contact, or what the process was - leading to back-and-forth emails and delays.

What I Built

01

Advanced Filtering System

Built a sidebar filter system with 6 distinct filter modes: Business Function, Business SME, IT Application Owner, Vendor, Business Capability, and Coverage Regions. Filters work together so users can combine criteria - for example, finding all finance apps available in Europe owned by a specific IT lead. All filters integrate with a live search bar and clear together when switching modes.

02

Recently Viewed History

Implemented a Recently Viewed feature backed by a dedicated SharePoint list (AFT Recently Viewed) that logs each user's app interactions with timestamps. The catalog surfaces your most recently accessed apps sorted by viewed date, making it fast to return to tools you use regularly.

03

Request Access Workflow

Designed a conditional access request system driven by a Request_Process field with 5 distinct scenarios: Saviynt form, SNOW ticket, direct email, instruction document, or other. Each app card and detail page surfaces the correct access path for that specific application - clicking a request email opens a pre-addressed email, clicking a Saviynt link opens the correct form, and instruction documents open the relevant guide. No more guessing how to get access.

04

Detailed Application Profiles

Each application has its own detail page showing a full description, primary and secondary business function classifications, geographic coverage (regional or global), IT and business owner contacts, and vendor information. Users can launch the app directly from the catalog or navigate to all access options from one screen.

Technical Details

Power Fx Formula Optimization

Written optimized Power Fx formulas to keep search and filtering responsive across 900+ records. Worked around Power Apps delegation limits by structuring queries carefully to avoid performance degradation at scale.

SharePoint Data Architecture

Backend uses two SharePoint lists: the main AFT catalog list with fields for multi-choice business functions, person fields for IT and business leads, hyperlink fields for access instructions, and choice fields for request process type. A second AFT Recently Viewed list tracks user history with timestamps for the Recently Viewed feature.

Conditional UI Logic

App cards and detail pages use conditional visibility logic tied to Request_Process values so the correct access buttons appear automatically for each application without any manual configuration per app.
